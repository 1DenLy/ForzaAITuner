# 04. Механика привязки данных (Data Binding)

Этот документ описывает техническое ядро UI-слоя — систему автоматической и безопасной двунаправленной привязки данных между виджетами Qt (PySide6) и Pydantic-моделями (`TuningSetup`).

## Суть и мотивация

В приложении существует более 50 параметров тюнинга. Если писать ручные обработчики `get()` и `set()` для каждого QSlider и QComboBox, код окон быстро станет нечитаемым и подверженным ошибкам. 

Вместо этого мы используем **декларативный подход**: описываем маппинг один раз в виде списка правил, а специальный класс `TuningMapper` автоматически:
1. Выполняет конвертацию типов (например, `int` из слайдера в `float` для модели).
2. Читает допустимые диапазоны и списки прямо из доменного слоя.
3. Проводит двунаправленный обмен данными (`UI -> Model` и `Model -> UI`).

---

## 1. WidgetBinding (Правило привязки)

`WidgetBinding` — это `dataclass`, который связывает имя виджета в Qt Designer с путём к полю в Pydantic модели.

```python
@dataclass
class WidgetBinding:
    widget_name: str                     # Имя виджета (например, "cb_road_type")
    model_path: tuple[str, ...]          # Путь к полю (например, ("session", "road_type"))
    ui_to_model: Callable[[Any], Any]    # Функция конвертации: UI int -> Model float
    model_to_ui: Callable[[Any], Any]    # Функция конвертации: Model float -> UI int
    label_name: Optional[str] = None     # Имя связанного QLabel (для показа текущего значения)
    label_format: str = "int"            # Формат вывода в QLabel ("int" или "float")
```

Определяется список таких привязок в `TuningMapper`. Путь `model_path` реализован кортежем строк для безопасного обхода вложенных словарей (`dict.get("session").get("road_type")`).

---

## 2. TuningMapper (Оркестратор)

`TuningMapper` — это основной инструмент синхронизации, который содержит карту всех `WidgetBinding` и реализует общую логику:

*   **`export_to_ui(model_data, ui_widgets)`**: Итерируется по привязкам, берёт значение из `model_data` (результат `model_dump()`), пропускает через `model_to_ui` и кладёт в виджет.
*   **`update_from_ui(ui_widgets)`**: Считывает значения с виджетов, пропускает через `ui_to_model` и собирает вложенный словарь для создания/обновления Pydantic-модели.
*   **Чтение/Запись виджетов**: Используется **паттерн "Стратегия" (Strategy)** с реестрами `extractors` и `injectors`. Выбор стратегии идет как по точному типу, так и с учетом наследования (MRO) через `isinstance`. При инициализации мапперу передаются (или создаются по умолчанию) словари, где ключ — тип виджета (например, `QCheckBox`), а значение — функция-лямбда для извлечения/установки значения. Это защищает код от нарушения *Open/Closed Principle* — новые кастомные виджеты могут быть добавлены снаружи через `register_widget_type()` без изменения ядра маппера.

Дополнительные обязанности маппера:
*   Синхронизация слайдеров и их `QLabel` (показ актуального значения при перетягивании ползунка).
*   Подсветка виджетов красным при ошибках валидации (через CSS/QSS-свойства `error`).

---

## 3. Чтение ограничений из Pydantic (TuningDefaults)

Чтобы не дублировать диапазоны слайдеров в UI, `TuningMapper` автоматически запрашивает лимиты у доменного слоя:

*Механизм `TuningDefaults.get_range(model_path)`:*
1. Принимает путь к полю (например, `("tires", "front_pressure_bar")`).
2. Опускается по иерархии Pydantic v2-моделей.
3. Достаёт метаданные Pydantic (ограничения, заданные через `Field(ge=1.0, le=3.8)`).
4. Возвращает кортеж `(lo, hi)`.

`TuningMapper.configure_ranges()` вызывает этот метод при старте окна, пересчитывает диапазоны через функции-трансформеры (чтобы адаптировать `float` к `int` диапазону Qt) и устанавливает `setMinimum`/`setMaximum` в слайдеры.

---

## 4. Трансформеры (Transformers)

PySide6 слайдеры (`QSlider`) поддерживают только целые числа (`int`), в то время как доменная логика оперирует дробными (`float`). Кроме того, некоторые строковые данные в UI (например, из `QComboBox` или `QCheckBox`) могут требовать преобразования в `bool`.

Для устранения этого разрыва используется модуль `transformers.py` с функциями-конвертерами:

*   **Фабрика множителей (`_make_pair`)**:
    В коде реализована фабрика `_make_pair(factor)`, чтобы не писать функции под каждый абстрактный масштаб (×10, ×100 и т.д.). Она динамически генерирует пару функций `(to_model, to_ui)` для заданного масштаба.
*   **Множитель ×10** (шаг 0.1): 
    Слайдер выдает 21 → `slider_x10_to_model` → В модель идет `2.1`
    В модели `2.1` → `slider_x10_to_ui` → Слайдер ставится на `21`.
*   **Множитель ×1** (шаг 1.0):
    Аналогично, но без смещения разряда.
*   **Булевые трансформеры (`str_to_bool` / `bool_to_str`)**:
    Паттерн для привязки логических значений (например, из текстовых элементов `QComboBox` или строковых состояний `QCheckBox`).
    `"True"` (строка из UI) → `str_to_bool` → `True` (булево значение в модели).
    `False` (булево из модели) → `bool_to_str` → `"False"` (строка в UI).

Функции покрыты логированием: при передаче неверных данных они не роняют приложение, а возвращают безопасное значение по умолчанию (например, `0.0` или `0`) и пишут `warning` в лог.

## 5. Допустимые значения (TuningOptions)

Для выпадающих списков (`QComboBox`) UI-слой не хранит список допустимых вариантов ("Dry", "Wet", "Snow / Ice").

Эту информацию предоставляет доменный класс `TuningOptions`.
`TuningMapper.populate_combo_boxes()`:
1. Проходится по всем виджетам типа `QComboBox`.
2. Смотрит их `model_path`.
3. Запрашивает через `TuningOptions.for_path(model_path)`.
4. Если есть варианты, вызывает `widget.addItems()`.

**Итог архитектуры:** При добавлении нового параметра тюнинга достаточно добавить поле в Pydantic и одну строчку `WidgetBinding` в `TuningMapper`. UI-диапазоны и ограничения подтянутся автоматически.

## 6. Безопасность и логирование (Security & Logging)

При обработке данных в мапперах и трансформерах действуют строгие правила безопасности:

1.  **Отказоустойчивость (Fail-Soft):** Если трансформер получает некорректные данные, он обязан вернуть безопасное значение по умолчанию (например, `0` или пустую строку) и записать `warning` в лог, чтобы не прерывать работу интерфейса.
2.  **Защита приватных данных (PII/Secret Protection):** При логировании ошибок маппинга или валидации **категорически запрещено** выводить в лог значения приватных полей. Сюда относятся:
    *   Токены авторизации (JWT, API Keys) из `SettingsDialog`.
    *   Пароли или персональные данные пользователя.
    *   Любые поля, помеченные в модели как чувствительные.
    *   *Пример корректного лога:* `WARNING: Failed to map 'auth_token' - invalid format` (без указания самого значения токена).
