# ПЛАН ВНЕДРЕНИЯ ЗАЩИТЫ ОТ PATH TRAVERSAL В ИНФРАСТРУКТУРЕ

## Цель
Обеспечить безопасность файловых операций в `LocalConfigRepository`, используя стандартизированные валидаторы проекта.

## Шаги
1. **Инжекция валидатора**: Обновить конструктор `LocalConfigRepository`, чтобы он принимал `PathValidator`.
2. **Делегирование проверки**: Заменить ручную проверку `is_relative_to` в методе `_get_safe_path` на вызов `path_validator.validate()`.
3. **Обработка ошибок**: Обеспечить выброс `SecurityViolationError` (или аналогичного исключения инфраструктурного слоя) при провале валидации.
4. **Поддержка динамических путей**: (Опционально) Расширить методы `load`/`save` для поддержки путей, переданных извне (для экспорта/импорта), с их обязательной валидацией.
5. **Тестирование**: Добавить/обновить тесты, подтверждающие блокировку путей вне разрешенной директории.
